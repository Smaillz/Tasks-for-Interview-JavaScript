1. Каково назначение обертывания всего содержимого исходного файла JavaScript в функциональном блоке?
;(function(param) { /* some code */ } )(param);

Отв: Это все более распространенная практика, используемая многими популярными библиотеками JavaScript (jQuery, Node.js и т. Д.). 
Этот метод создает закрытие всего содержимого файла, что, возможно, самое важное, создает частное пространство имен и 
тем самым помогает избежать потенциальных конфликтов имен между различными модулями JavaScript и библиотеками.
Точка с запятой, в начале, чтобы не повалился скрипт, тк в js не обязательно ее ставить, и поставив ее вначале файла гарантируем что не будет ошибок связанных с ней.
